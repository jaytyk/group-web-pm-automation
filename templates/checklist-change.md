## ✅ PM 단계별 체크리스트 (운영 변경)

### 0. 접수/분류
- [ ] 변경유형/영향범위(페이지·기능·인프라) 확정
- [ ] 긴급도/릴리즈 희망일 확인
- [ ] 리스크(PII/SEO/배포/장애 가능성) 판단

### 1. 요구사항 명세
- [ ] AS-IS/TO-BE 명확화(스크린샷/링크)
- [ ] 수용기준(AC) 정의
- [ ] 관련 부서 승인(정책/법무/보안) 필요 여부 확인

### 2. 구현/검증
- [ ] 개발 범위/일정/담당자 확정
- [ ] QA 시나리오/회귀 범위 정의
- [ ] 배포 창/롤백 계획 수립

### 3. 릴리즈/공지/운영
- [ ] 릴리즈 노트/공지 필요 여부
- [ ] 모니터링(에러/트래픽/전환) 확인 항목
- [ ] 완료 보고 및 이슈 종료
