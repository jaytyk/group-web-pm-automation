## ✅ PM 단계별 체크리스트 (신규 구축)

### 0. 접수/트리아지
- [ ] 요청 범위/목표 1문장 정리
- [ ] 이해관계자/승인자(RACI) 확정
- [ ] 일정 기대치(희망 오픈일/이벤트일) 확인
- [ ] 리스크(PII/로그인/외부연동/법무) 플래그

### 1. 요구사항/정책 확정
- [ ] 사이트맵(IA) 초안
- [ ] 콘텐츠 목록/원고/번역 범위 확정
- [ ] 운영정책(게시/수정 권한, 게시 프로세스)
- [ ] 개인정보/약관/쿠키/추적(GA) 정책 확인

### 2. 설계/디자인
- [ ] 와이어프레임/UX 시나리오
- [ ] 디자인 가이드/컴포넌트 합의
- [ ] 접근성/반응형 기준 합의

### 3. 개발/인프라
- [ ] 기술 스택/배포 방식 확정
- [ ] 도메인/DNS/SSL/CDN 계획
- [ ] 로그/모니터링/알림 기준

### 4. QA/보안/법무(필요 시)
- [ ] QA 범위/시나리오/기기·브라우저 매트릭스
- [ ] 보안 점검(OWASP, 헤더, 취약점)
- [ ] 개인정보 영향 검토(해당 시), 처리위탁/보관기간/파기

### 5. 오픈/운영이관
- [ ] 리허설(스테이징→프로덕션)
- [ ] 롤백 플랜
- [ ] 운영 매뉴얼/계정/권한 이관
- [ ] 종료 조건(오픈 확인, 모니터링 24~72h)
